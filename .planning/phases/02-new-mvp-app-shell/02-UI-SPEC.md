## Typography

**Fonts must be loaded via Next.js `next/font/google` with `display: swap`.**

### Type Scale

| Role | Font | Size | Weight | Line Height | Letter Spacing | Transform |
|------|------|------|--------|-------------|----------------|-----------|
| Display | Space Grotesk | 48px | 300 (thin) | 1.1 | +8% (0.08em) | none |
| Heading | Space Grotesk | 28px | 300 (thin) | 1.2 | +5% (0.05em) | none |
| Body | Manrope | 16px | 500 (medium) | 1.5 | 0 | none |
| Label | Manrope | 12px | 500 (medium) | 1.4 | +8% (0.08em) | uppercase |

### Typography Rules

- **Hierarchy is extreme:** Use Display (48px) for the primary focal point per screen, Label (12px) for all secondary metadata, chips, step indicators, and form labels. Avoid the 16–28px "medium zone" for structural hierarchy — it flattens the editorial feel.
- **Weights declared:** 300 (thin) for Space Grotesk display and headings only. 500 (medium) for all Manrope body and labels. These are the only 2 weight values in the system — no weight 400 anywhere.
- **No medium-sized text for hierarchy:** If something needs to feel important, make it large. If it's supporting info, make it small. Never bump body text to 18px to signal importance.
- **Label style is non-negotiable:** All form labels, chips, metadata, and step indicators use Label role (12px Manrope 500, uppercase, +8% tracking). Always visible above the input — never as placeholder text.
- **Section headers that previously used Subheading (20px)** are remapped: use Heading (28px) if it is the primary title of a screen or section, or Body (16px) if it is supporting prose. No 20px size exists in the system.

## Color

**Source: Stitch design session / Orbital Drift system. All values override shadcn slate defaults via CSS variable reassignment in `globals.css`.**

### Token Map

| Role | Hex | HSL (for CSS var) | Usage |
|------|-----|-------------------|-------|
| Dominant surface (60%) | `#11131a` | 228 22% 8% | Page background, full-bleed void areas |
| Surface tier 1 | `#0c0e14` | 228 26% 6% | Deepest inset surfaces, bottom nav bg |
| Surface tier 2 | `#1a1d26` | 228 20% 12% | Cards, glass containers (pre-blur base) |
| Surface tier 3 | `#33353c` | 228 8% 21% | Elevated cards, modal backgrounds |
| Secondary (30%) | `#374956` | 207 20% 27% | Sidebar tones, atmospheric depth accents, teal-navy passages |
| Secondary light | `#b6c9d9` | 207 25% 78% | Secondary text, atmospheric indicators, stat labels |
| Accent cream-gold (10%) | `#E8D5A3` | 41 57% 78% | Reserved list — see below |
| Accent cream-gold bright | `#fff2d4` | 41 100% 91% | Hover state of cream-gold elements only |
| Mark purple | `#9b7fe8` | 258 65% 71% | Mark message indicator dot / reply attribution exclusively |
| Destructive | `#ef4444` | 0 84% 60% | Destructive actions only (disconnect account) |
| Text primary | `#f0ece0` | 41 33% 91% | Primary readable text on dark surfaces |
| Text secondary | `#8892a0` | 213 14% 57% | Supporting metadata, timestamps, secondary labels |

### Accent Reserved For

The cream-gold accent (`#E8D5A3` / `#fff2d4`) is reserved EXCLUSIVELY for:

1. Primary CTA buttons (fill color)
2. Active/selected navigation state indicator
3. Data points in the stats card (ER %, follower count, post count)
4. The gold send button in Mark Chat
5. ER projection pill in Edit Feedback ("Estimated: 4.6–4.9% ER with edit")
6. "MARK ACTIVE" indicator dot / status signal
7. The brightest star visual in the galaxy (best-performing post)

Accent must NOT appear on: hover states of non-CTA elements, borders, decorative lines, or text outside the above list.

### Color Application Rules

- **No 1px solid borders.** Separation is achieved via tonal surface shifts and ambient shadow depth — never a visible border line.
- **Glassmorphism surfaces:** `backdrop-filter: blur(20px)`, background at 40–60% opacity of the surface tier 2 base, plus a 1px ghost border at `rgba(255,255,255,0.08)` — this ghost border counts as the one structural exception to the no-border rule.
- **Orbit decorative lines:** `outline-variant` at 10% opacity (`rgba(255,255,255,0.10)`). These are decorative only — not containers.
- **Ambient glow (not shadow):** `box-shadow: 0 0 60px rgba(55,73,86,0.05)` — tinted with secondary at 5%, blur 60px+. This is a glow, not a drop shadow.
- **Cards use no horizontal dividers.** 16–24px vertical whitespace only between card sections.
- **Dark mode only.** The app is dark-mode-first and only. No light mode variant.

## Component Patterns

These prescriptions apply to all components built in this phase. They supplement shadcn's Radix primitives.

### Inputs

- Style: underline-only. Single 1px line at bottom. No box, no border-radius, no background fill.
- CSS: `border: none; border-bottom: 1px solid rgba(255,255,255,0.20); border-radius: 0; background: transparent;`
- Focus state: bottom border brightens to `rgba(232,213,163,0.60)` (cream-gold at 60% opacity). No outline ring.
- Placeholder text: `rgba(136,146,160,0.50)` — secondary text at 50% opacity. Never used as a label.
- Label: always rendered as a separate Label element above the input using the Label type role (12px Manrope 500 uppercase).

### Buttons — Primary

- Background: `#E8D5A3` (cream-gold)
- Text: `#11131a` (void background — high contrast)
- Border-radius: fully rounded (`9999px`)
- Border: none
- Height: 44px minimum (touch target)
- Padding: 16px horizontal
- Font: Manrope 500, 12px, uppercase, +6% tracking
- Hover: background transitions to `#fff2d4` (cream-gold bright), transition: 150ms ease

### Buttons — Secondary

- Background: `rgba(51,53,60,0.50)` (surface tier 3 at 50%)
- Border: 1px solid `rgba(255,255,255,0.12)` (ghost border — structural exception)
- Border-radius: fully rounded (`9999px`)
- Text: `#f0ece0` (text primary)
- Height: 44px minimum
- Font: Manrope 500, 12px, uppercase, +6% tracking
- Hover: background transitions to `rgba(51,53,60,0.80)`, transition: 150ms ease

### Cards / Glass Containers

- Background: `rgba(26,29,38,0.60)` — surface tier 2 at 60% opacity
- Backdrop filter: `blur(20px)`
- Ghost border: `1px solid rgba(255,255,255,0.08)`
- Border-radius: 16px
- No internal horizontal dividers
- Vertical spacing between card sections: 16–24px

### Mark's Message Indicator

- A 6px circle, `background: #9b7fe8` (Mark purple)
- Positioned to the left of Mark's message bubble / text block
- Used EXCLUSIVELY to attribute Mark's responses in chat

## Screen-Specific Contracts

### Galaxy Home (Primary Home Screen)

- Planet/world: centered, minimum 320px diameter, subtly animated (slow rotation via CSS animation or Nano Banana 2)
- "MARK ACTIVE" indicator: top-left, Label role "MARK ACTIVE" + 6px cream-gold dot
- Stats card (glass): ER %, followers, posts — all data values in cream-gold accent; labels in secondary text color (Label role)
- "TALK TO MARK" button: Primary button style, centered below planet
- "SNAPSHOT STARTER" entry: Secondary button style, bottom of screen
- Galaxy background: deep void `#11131a`, ambient glow using secondary tint at 5% blur 60px+
- Posts as stars: if post data is available, render stars at positions relative to planet. ER as brightness (CSS `opacity` + `filter: brightness()`). Best post = brightest, closest to center. If no posts yet: empty state (see copywriting).

### Mark Chat Screen

- Header: "MARK · ARTIST INTELLIGENCE" — Heading role (Space Grotesk 28px 300, uppercase, +5% tracking)
- No tier number shown. No "MARK IS WATCHING" text. Mark's contextual intelligence is invisible.
- Mark messages: left-aligned, preceded by 6px purple dot
- User messages: right-aligned, glass card style
- Input: underline-style, full-width, placeholder "Ask Mark anything..."
- Send button: gold (cream-gold), circular, 44px, arrow-up icon

### Onboarding Flow (3 steps)

- Step indicator: "THE MULTIVERSE · 01/03" — Label role, top of screen
- Step 1 (Sign Up): email + password, underline inputs, primary CTA "CREATE ACCOUNT"
- Step 2 (Instagram Connect): "Connect Your World" heading (Heading role), Instagram handle input (underline), "CONNECT INSTAGRAM" primary CTA, "Business or Creator account required" subtext in Body role, secondary text color
- Step 3 (Scrape Loading): animated galaxy/loading state, "MARK IS READING YOUR WORLD" label (Label role), progress indication
- Transitions between steps: Nano Banana 2 fade+slide (200ms ease)

### Edit Feedback

- Header: "EDIT FEEDBACK · @{handle}" — Heading role (Space Grotesk 28px 300)
- Video preview card: glass card with timeline chip (duration) + ER chip — both in Label role
- Mark's feedback: Body role, preceded by purple dot
- ER projection pill: cream-gold accent, "Estimated: X.X–X.X% ER with edit" — Label role inside a fully-rounded pill
- Input options: "PASTE INSTAGRAM URL" underline input + "UPLOAD VIDEO" primary button
- Ongoing relationship framing: header/subtext should feel like a recurring session, not a one-off audit

### Snapshot Starter

- URL paste input: underline style, full-width
- Reference cards: glass card, grow as URLs are added, "ANALYZING..." state with subtle pulse animation
- Analysis chips on reference cards: Label role, secondary surface background
- "GENERATE PLAN" CTA: primary button
- Generated plan: Mark's summary (Body role), idea cards (glass), ER projections (cream-gold)
- "ADD TO CALENDAR" + "TALK TO MARK ABOUT THIS PLAN": secondary and primary buttons respectively

## Accessibility Baseline

- Minimum contrast ratio: 4.5:1 for body text (`#f0ece0` on `#11131a` = 15.3:1 — passes AAA)
- Minimum contrast ratio for accent on void: `#E8D5A3` on `#11131a` = 9.8:1 — passes AAA
- Minimum touch target: 44px (all interactive elements)
- Focus indicators: cream-gold ring at 2px offset for keyboard navigation (replace shadcn default ring with `outline: 2px solid rgba(232,213,163,0.80); outline-offset: 2px`)
- Screen reader: all icon-only buttons must have `aria-label`. Mark's purple dot must have `aria-label="Mark's message"` or be `aria-hidden` if attribution is in adjacent text.
- Reduced motion: all Nano Banana 2 animations must respect `prefers-reduced-motion` — reduce to fade-only, no translation or scale.
